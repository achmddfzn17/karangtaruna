📋 Keterangan Tabel dan Relasi

auth_users (Supabase Built-in)
Tabel autentikasi bawaan Supabase. Menyimpan kredensial login pengguna.
profiles (Extends auth_users)
Data profil pengurus/pengguna sistem. Relasi 1:1 dengan auth_users.
RLS Policy: Admin dapat CRUD semua; Member hanya READ profil sendiri.
members
Data lengkap keanggotaan Karang Taruna (termasuk anggota non-pengguna sistem).
RLS Policy: Admin dapat CRUD; Member dapat READ semua.
transaction_categories
Kategori transaksi keuangan (Pemasukan/Pengeluaran).
RLS Policy: Admin dapat CRUD; Member dapat READ.
transactions
Rekaman transaksi keuangan kas organisasi.
Relasi: Many-to-One dengan transaction_categories dan profiles (created_by).
RLS Policy: Admin dapat CRUD; Member dapat READ semua.
activities
Data kegiatan dan program kerja organisasi.
Relasi: Many-to-One dengan profiles (created_by).
RLS Policy: Admin dapat CRUD; Member dapat READ + INSERT participants.
activity_documents
Dokumentasi foto/video/file kegiatan (disimpan di Supabase Storage).
Relasi: Many-to-One dengan activities dan profiles (uploaded_by).
RLS Policy: Admin dapat CRUD; Member dapat INSERT untuk kegiatan sendiri.
activity_participants
Daftar peserta kegiatan dengan status kehadiran.
Relasi: Many-to-One dengan activities dan members.
RLS Policy: Admin dapat CRUD; Member dapat READ kegiatan terkait.
⚠️ Catatan Penting:
• Semua tabel menggunakan UUID sebagai Primary Key
• Row Level Security (RLS) diterapkan pada semua tabel untuk keamanan data
• File media (foto, video, dokumen) disimpan di Supabase Storage, hanya URL yang disimpan di database
• Relasi Foreign Key menggunakan ON DELETE CASCADE untuk integritas referensial
• Timestamp created_at dan updated_at otomatis dikelola oleh Supabase

🔐 Row Level Security (RLS) Policies

Admin (role: ADMIN, SUPER_ADMIN):

  • Full CRUD access pada semua tabel
  • Dapat mengelola data pengguna, keuangan, kegiatan, dan dokumentasi

Member (role: ANGGOTA):

  • READ access pada sebagian besar tabel
  • Dapat mendaftar sebagai peserta kegiatan (INSERT activity_participants)
  • Dapat upload dokumentasi untuk kegiatan yang diikuti
  • Hanya dapat melihat dan edit profil sendiri

📊 Kardinalitas Relasi

  • auth_users ↔ profiles: One-to-One (1:1)
  • profiles → transactions: One-to-Many (1:N) - Satu user dapat membuat banyak transaksi
  • profiles → activities: One-to-Many (1:N) - Satu user dapat membuat banyak kegiatan
  • profiles → activity_documents: One-to-Many (1:N) - Satu user dapat upload banyak dokumen
  • transaction_categories → transactions: One-to-Many (1:N) - Satu kategori untuk banyak transaksi
  • activities → activity_documents: One-to-Many (1:N) - Satu kegiatan memiliki banyak dokumen
  • activities → activity_participants: One-to-Many (1:N) - Satu kegiatan memiliki banyak peserta
  • members → activity_participants: One-to-Many (1:N) - Satu anggota dapat ikut banyak kegiatan
